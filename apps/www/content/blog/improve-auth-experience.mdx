# 改进认证体验

// 背景说明
我们一直在努力改进 Unkey 的认证体验。今天，我们很高兴分享我们在这方面所做的改进。这些改进涉及面广泛 - 从支持无密码登录到改进 CLI 认证流程。

## 无密码登录 Passwordless Login

// 认证流程说明
在 Unkey，我们采用了现代的无密码认证方法。不再需要记住复杂的密码，而是使用以下简单流程：

1. 输入您的电子邮件地址
2. 点击发送到您邮箱的魔术链接
3. 完成！您已经登录了

// 技术实现
这个流程背后采用了最新的认证技术，包括:
- 安全的一次性令牌生成
- 防重放攻击保护
- 链接的有限时效性

## CLI 认证改进 CLI Authentication

// CLI 认证流程优化
为了让开发者能更轻松地使用我们的 CLI 工具，我们重新设计了认证流程：

```bash
unkey login
```

// 新流程说明
运行此命令会：
1. 打开浏览器进行认证
2. 自动完成令牌交换
3. 配置 CLI 凭据

不再需要手动复制粘贴令牌或处理复杂的配置文件。

## 工作区切换 Workspace Switching

// 多工作区管理
对于在多个组织中工作的用户，我们简化了工作区切换流程：

```bash
unkey switch
```

// 交互式选择
这个命令会显示一个交互式列表，让您可以轻松切换不同的工作区。

## API 认证 API Authentication

// API 访问认证
对于通过 API 访问 Unkey 的用户，我们提供了两种认证方式：

1. **根密钥 (Root Keys)**
   // 根密钥使用场景
   - 用于管理操作
   - 完整的 API 访问权限
   - 支持精细的权限控制

2. **访问令牌 (Access Tokens)**
   // 访问令牌特点
   - 临时访问
   - 有限的权限范围
   - 自动过期机制

## 安全性改进 Security Improvements

// 安全增强措施
我们还实施了多项安全改进：

- **设备追踪 Device Tracking**
  // 设备管理功能
  查看和管理已认证的设备

- **会话管理 Session Management**
  // 会话控制
  随时查看和撤销活动会话

- **审计日志 Audit Logs**
  // 安全审计
  记录所有认证相关的活动
